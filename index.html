DirectX Project

This project was apart of my coursework at University.
In DirectX11 our task was to create the graphics for a game engine and the basics using C++ and DirectX 11. This entailed making the program to have phong lighting, texturing and game controls.
However as I quickly added these in, I started adding in additional features.
All of the features that I included in this project:

  • Skybox
  • Billboards
  • Specular mapping
  • Normal mapping
  • Post Processing
  • Keyboard and Xbox360 controller support using XInput
  • Directional, Point and Spot light
  • OBJ file Loading
  • Scene graph loading and saving using Jsons
  • Tessellating Water

In my last Project, Modern OpenGL, I had create some simple animated water, but now I had more of an understanding of how shaders work and how to use them. I choose to create water once more, but this time using hull, domain and geometry shaders. This allowed me to create more complex looking water than before. I was also able to move it so that further you are from the water, the less triangles it would add to the surface of the water.
I also used libraries such as Imgui for the UI, and UI and model assets from Kenny.

  • Completed Date: 1st September 1970
  • Grade: 100%

Modern OpenGL 3.3

This project was apart of my coursework at University.
We was give the task to make a 3D game of our choice with FreeGlut, which is an old toolkit for old OpenGL. I quickly completed this work and after talking to my tutor I moved on to Modern OpenGL 3.3. With this new API and using the knowledge I quickly gained from FreeGlut I built this game.
I wanted to play around with world generation and so I added in a seed based world generator; however, in retrospect I feel I could of improved it greatly. I was also intrested in Water and how it works, so I started working on water and in the end using the vertex shader I was able to create this simple wave animation for the water.
All of the features that I included in this project:

  • Water animation using the vertex shader
  • World Generation
  • Day and night cycle
  • File saving and loading
  • Object Editor
  • OBJ file reader
  • Primitive Cube map

Due to time constrains, I wasn't able to add being able to place your own buildings in to the world that you make in the editor. I also used libraries such as Imgui for the UI, and model assets from Kenny.

  • Completed Date: 21st April 2017
  • Grade: 99%

Koin!

Koin is my first released game on the Google PlayStore. Koin started as a simple 2D game, and I went on to making it 3D with Unity. Point of the game is to collect coins and upgrade yourself, and collect more coins. I went through the process of designing, programming and releasing this game. I learnt a lot about the development process.
As I am writing this I feel I can improve it greatly as it was released in May of 2016. Some improves would deal with the performace issues, look and style of the game. As Unity has updated to improve its support of 2D games, I might think of bring it back to a 2D style.
